# 사용자 보안 가이드

> [← 운영](../operations-guide.md) | [보안](./hardening.md) | [사용자 보안](./user-security.md)

## 개요

**사용자(사람)는 종종 보안 체인에서 가장 약한 링크입니다.**  
사용자 보안을 강화하여 전체 보안 체인을 보호합니다.

> **사용자 보안 원칙**:  
> - 강력한 비밀번호 정책
> - 사용자 교육
> - 행동 모니터링
> - 감사 로깅
> - 접근 제어

---

## 비밀번호 정책

### 현재 정책

**최소 요구사항**:
- 최소 길이: **12자** (강력한 최소값)
- 대문자: 필수
- 소문자: 필수
- 숫자: 필수
- 특수문자: 필수

**추가 검증**:
- 일반적인 약한 비밀번호 차단
- 문자 유형 다양성 검사 (3가지 이상)

### 비밀번호 예시

**✅ 강력한 비밀번호**:
- `MyP@ssw0rd2024!`
- `Tr0ub@dor&3`
- `C0mpl3x!P@ss`

**❌ 약한 비밀번호**:
- `password123` (너무 일반적)
- `12345678` (숫자만)
- `qwerty` (너무 짧음)

---

## 계정 잠금 정책

### 실패 시도 제한

**설정**:
- 최대 실패 시도: **5회**
- 잠금 시간: **15분**

**동작**:
1. 5회 실패 시 계정 잠금
2. 15분 후 자동 해제
3. 모든 실패 시도 로깅

---

## 세션 관리

### JWT 토큰

**설정**:
- 만료 시간: **24시간**
- 서명 알고리즘: **Ed25519** (하드웨어 최적화)
- 토큰 검증: 모든 요청에서 검증

**보안 기능**:
- 토큰 만료 자동 처리
- 서명 검증
- 역할 기반 접근 제어

---

## 접근 제어

### 역할 기반 접근 제어 (RBAC)

**역할**:
- **Admin**: 모든 권한
- **User**: 제한된 권한

**권한 검사**:
- 모든 보호된 엔드포인트에서 역할 검사
- 리소스 소유권 확인
- 최소 권한 원칙 적용

---

## 감사 로깅

### 로깅되는 사용자 행동

**인증 관련**:
- 로그인 성공/실패
- 로그아웃
- 토큰 갱신

**리소스 접근**:
- VM 생성/삭제
- 사용자 관리
- 설정 변경

**모든 행동**:
- 사용자 ID
- IP 주소
- 타임스탬프
- 성공/실패 여부

---

## 사용자 교육

### 보안 정책 문서

사용자는 다음을 이해해야 합니다:

1. **강력한 비밀번호 사용**
   - 최소 12자
   - 복잡한 문자 조합
   - 정기적 변경 (90일)

2. **피싱 방지**
   - 의심스러운 이메일 확인
   - 링크 클릭 전 확인
   - 비밀번호 공유 금지

3. **세션 관리**
   - 공용 컴퓨터에서 로그아웃
   - 장시간 비활성 시 재로그인

4. **보고 의무**
   - 이상 행동 발견 시 즉시 보고
   - 보안 사고 신고

---

## 행동 모니터링 (향후 구현)

### 이상 행동 감지

**감지 항목**:
- 다른 IP에서의 로그인
- 비정상적인 시간대 접근
- 권한 상승 시도
- 대량 데이터 접근
- 빠른 연속 요청

**대응**:
- 자동 알림
- 계정 일시 잠금
- 관리자 알림

---

## MFA (Multi-Factor Authentication) (향후 구현)

### 계획된 기능

**인증 방법**:
- TOTP (Time-based One-Time Password)
- SMS 코드
- 하드웨어 토큰

**적용 범위**:
- 관리자 계정 필수
- 일반 사용자 선택
