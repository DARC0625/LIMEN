# 왜 프론트엔드/백엔드 서버에 docs/ 폴더가 없는가?

## 📋 개요

프론트엔드와 백엔드 서버에서는 **docs/ 폴더를 체크아웃하지 않습니다**. 대신 **RAG/ 폴더만** 사용합니다.

## 🎯 이유

### 1. RAG/ 폴더가 충분함

- **RAG/**: RAG 시스템용 구조화된 문서 (Git 추적)
- **docs/**: 원본 문서 (Git 추적)

프론트엔드와 백엔드 서버에서는 RAG 시스템을 통해 문서를 참조하므로, RAG/ 폴더만 있으면 충분합니다.

### 2. 불필요한 파일 제거

- docs/ 폴더는 모든 문서를 포함 (프론트+백엔드+운영 등)
- 각 서버에서는 필요한 문서만 RAG/에 구조화되어 있음
- 불필요한 파일을 다운로드하지 않아 효율적

### 3. 명확한 역할 분리

```
프론트엔드 서버:
├── frontend/     ✅ 작업 영역
├── RAG/          ✅ 문서 참조 (RAG 시스템)
└── docs/         ❌ 불필요 (RAG/에 포함됨)

백엔드 서버:
├── backend/      ✅ 작업 영역
├── RAG/          ✅ 문서 참조 (RAG 시스템)
└── docs/         ❌ 불필요 (RAG/에 포함됨)
```

## 🔄 RAG/ 폴더의 역할

RAG/ 폴더는 다음을 포함합니다:

- `01-architecture/` - 아키텍처 문서
- `02-development/` - 개발 가이드 (프론트+백엔드)
- `03-deployment/` - 배포 가이드
- `04-operations/` - 운영 가이드
- `05-frontend/` - 프론트엔드 관련 문서
- `99-archive/` - 아카이브 문서

이 모든 문서는 프론트엔드와 백엔드 서버에서 **동일하게** 유지됩니다.

## 📚 문서 서버 (RAG 시스템)

문서 서버에서는 docs/ 폴더만 체크아웃합니다:

```bash
# 문서 서버 설정
git sparse-checkout set docs/
```

문서 서버는:
- RAG 시스템에 문서를 인덱싱
- docs/ → RAG/ 변환
- 벡터 DB 업데이트

## ✅ 결론

**프론트엔드와 백엔드 서버에는 docs/ 폴더가 필요 없습니다.**

- ✅ RAG/ 폴더만 있으면 충분
- ✅ RAG/는 프론트엔드와 백엔드 서버에서 동일
- ✅ AI는 RAG/를 통해 모든 문서 참조 가능
- ✅ 불필요한 파일 다운로드 방지
