# LIMEN 프로젝트 마크다운 문서 작성 가이드

## 📋 문서 작성 규칙

### 1. 문서 위치 규칙

모든 마크다운(.md) 파일은 **반드시 `RAG/` 폴더 내**에 위치해야 합니다.

#### 허용되는 위치
- ✅ `RAG/` 폴더 내의 모든 하위 디렉토리
- ✅ 각 폴더의 `README.md` (해당 폴더 설명용)

#### 금지되는 위치
- ❌ 프로젝트 루트에 MD 파일 생성 (README.md 제외)
- ❌ `backend/`, `frontend/` 등 코드 폴더에 MD 파일 생성
- ❌ `scripts/`, `config/` 등 설정 폴더에 MD 파일 생성

### 2. 문서 분류 구조

```
RAG/
├── 00-home.md                    # 프로젝트 홈
├── 01-architecture/              # 아키텍처 문서
├── 02-development/               # 개발 가이드
├── 03-deployment/                 # 배포 가이드
├── 04-operations/                 # 운영 가이드
│   ├── reports/                   # 보고서
│   ├── troubleshooting/           # 문제 해결
│   └── ...
├── 05-frontend/                   # 프론트엔드 문서
└── 99-archive/                    # 아카이브 (구버전/레거시)
```

### 3. 언어 규칙

**모든 마크다운 문서는 한글로 작성해야 합니다.**

#### 필수 사항
- ✅ 제목, 설명, 내용 모두 한글로 작성
- ✅ 코드 주석도 가능한 한글로 작성
- ✅ 기술 용어는 한글 병기 가능 (예: "WebSocket(웹소켓)")

#### 예외 사항
- 코드 예제의 실제 코드는 영어로 작성
- API 엔드포인트, 변수명, 함수명 등은 영어로 작성
- 외부 라이브러리/프레임워크 이름은 원문 유지

### 4. 파일 명명 규칙

#### 파일 이름
- 소문자와 하이픈(-) 사용
- 예: `backend-health-check.md`, `vm-console-fix.md`
- 한글 파일명은 피하세요 (인코딩 문제 가능)

#### 제목 구조
```markdown
# 문서 제목 (한글)

## 섹션 제목 (한글)

### 하위 섹션 (한글)
```

### 5. 문서 작성 템플릿

```markdown
# 문서 제목

## 개요
문서의 목적과 범위를 설명합니다.

## 내용
주요 내용을 작성합니다.

## 참고사항
추가 정보나 관련 문서 링크를 제공합니다.

## 관련 문서
- [관련 문서 1](./path/to/doc1.md)
- [관련 문서 2](./path/to/doc2.md)
```

### 6. 문서 이동/정리 규칙

#### 새 문서 생성 시
1. 적절한 `RAG/` 하위 폴더 선택
2. 파일명은 소문자+하이픈 형식
3. 내용은 한글로 작성
4. `RAG/README.md`에 링크 추가 (필요시)

#### 기존 문서 정리 시
- 루트나 코드 폴더에 있는 MD 파일은 `RAG/`로 이동
- 아카이브 문서는 `RAG/99-archive/`로 이동
- 보고서는 `RAG/04-operations/reports/`로 이동
- 문제 해결 가이드는 `RAG/04-operations/troubleshooting/`로 이동

### 7. README.md 규칙

각 폴더의 `README.md`는 해당 폴더의 설명을 담당합니다.

#### 위치별 역할
- **루트 `README.md`**: 프로젝트 전체 소개 및 빠른 시작
- **`RAG/README.md`**: 문서 인덱스 및 탐색 가이드
- **`backend/README.md`**: 백엔드 개발 가이드
- **`scripts/README.md`**: 스크립트 사용 가이드
- **`infra/README.md`**: 인프라 설정 가이드

### 8. 검증 체크리스트

문서 작성 후 다음을 확인하세요:

- [ ] 문서가 `RAG/` 폴더 내에 위치하는가?
- [ ] 내용이 한글로 작성되었는가?
- [ ] 파일명이 소문자+하이픈 형식인가?
- [ ] 제목과 섹션이 명확한가?
- [ ] 관련 문서 링크가 올바른가?

### 9. 자동 검증 (향후 구현)

프로젝트에 다음 검증 스크립트를 추가할 예정:
- MD 파일 위치 검증
- 한글 작성 여부 검증
- 파일명 규칙 검증
