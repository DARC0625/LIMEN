# 프론트엔드와 백엔드 서버 분리

## 중요: 서버 분리 구조

**프론트엔드 서버**:
- 경로: `/home/darc/LIMEN/frontend`
- 사용자: `darc`
- 서버: 프론트엔드 전용 서버
- PM2 프로세스: `limen-frontend`

**백엔드 서버**:
- 경로: `/home/darc0/LIMEN/backend` (또는 별도 서버)
- 사용자: `darc0` (또는 다른 사용자)
- 서버: 백엔드 전용 서버
- PM2 프로세스: `limen-backend` (백엔드 서버에서 실행)

## 작업 시 주의사항

1. **프론트엔드 작업 시**:
   - `/home/darc/LIMEN/frontend` 경로에서만 작업
   - 프론트엔드 서버의 PM2 로그만 확인 (`pm2 logs limen-frontend`)
   - 백엔드 파일은 참고용일 뿐, 직접 수정하지 않음

2. **백엔드 작업 시**:
   - 백엔드 서버에서 직접 작업
   - 프론트엔드 서버에서는 백엔드 로그 확인 불가
   - 백엔드 API 문서는 참고용

3. **GitHub 통합**:
   - GitHub는 통합 관리용
   - 프론트엔드와 백엔드 코드가 함께 있지만, 실제 서버는 분리됨

## API 통신

- 프론트엔드 → 백엔드: HTTP API 호출 (`/api/*` 경로)
- 프론트엔드 미들웨어가 `/api/*` 요청을 백엔드 서버로 프록시
- 백엔드 서버 IP: `10.0.0.100:18443` (기본값)

## 로그 확인

**프론트엔드 로그**:
```bash
pm2 logs limen-frontend
```

**백엔드 로그**:
- 백엔드 서버에서 직접 확인
- 프론트엔드 서버에서는 확인 불가

## 에러 처리

- 프론트엔드에서 백엔드 에러 발생 시:
  - 프론트엔드 콘솔에서 에러 메시지 확인
  - 백엔드 서버의 로그는 직접 확인 불가
  - 프론트엔드에서 상세 에러 로깅 추가 필요
