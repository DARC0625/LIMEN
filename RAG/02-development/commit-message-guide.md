# 커밋 메시지 가이드라인

## 원칙

**모든 커밋 메시지는 한글로 작성합니다.**

## 형식

```
<타입>: <한글 설명>

<상세 설명 (선택사항)>
```

## 타입

- `feat`: 새로운 기능 추가
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅, 세미콜론 누락 등 (기능 변경 없음)
- `refactor`: 코드 리팩토링
- `test`: 테스트 코드 추가/수정
- `chore`: 빌드 업무 수정, 패키지 매니저 설정 등
- `perf`: 성능 개선

## 예시

### 좋은 예시

```
feat: 자동 동기화 시스템 구축

- GitHub Actions 워크플로우 추가
- 백엔드/프론트엔드 서버 자동 동기화
- RAG 인덱싱 자동화
```

```
fix: 백엔드 서버 경로 수정

- /home/darc0/LIMEN으로 경로 수정
- 자동 동기화 정상 작동 확인
```

```
docs: RAG 폴더 구조 문서화

- RAG/ 폴더가 문서 폴더임을 명확히 함
- docs/ 폴더 제거 가이드 추가
```

### 나쁜 예시

```
fix: backend server path
```

```
feat: add auto sync
```

```
chore: update config
```

## 자동 커밋 스크립트

`scripts/auto-commit-push.sh`는 자동으로 한글 커밋 메시지를 생성합니다.

## 수동 커밋 시

```bash
git commit -m "feat: 새로운 기능 추가

상세 설명을 여기에 작성합니다."
```

## 주의사항

- 영어로 작성된 커밋 메시지는 피하세요
- 간결하고 명확하게 작성하세요
- 변경 이유를 설명하세요
