## 향후 계획

### 우선순위 1: 백업 자동화 스크립트 💾

**목표**: 데이터 손실 방지 및 빠른 복구

**작업 내용**:
- [ ] 데이터베이스 자동 백업 스크립트
- [ ] VM 이미지 백업 스크립트
- [ ] 백업 스케줄링 (cron)
- [ ] 백업 보관 정책 (30일, 90일 등)
- [ ] 백업 검증 스크립트
- [ ] 원격 백업 설정 (선택)
- [ ] 복구 테스트 시나리오

**예상 효과**:
- 데이터 안전성 보장
- 재해 복구 시간 단축
- 규정 준수

**기술 스택**:
- pg_dump (PostgreSQL)
- tar/gzip (파일 백업)
- cron (스케줄링)
- rsync (원격 백업)

**관련 문서**: [백업 가이드](../04-operations/backup/) (작성 예정)

### 우선순위 2: 통합 모니터링 서비스 구축 📊

**목표**: 모든 서비스를 통합 관리하는 중앙화된 모니터링 및 알림 시스템

**작업 내용**:
- [ ] 통합 모니터링 서비스 설계
- [ ] Prometheus + Grafana + Alertmanager 통합
- [ ] 각 서비스의 `/metrics` 엔드포인트 수집
- [ ] 알림 채널 설정 (이메일, Slack/Discord)
- [ ] 알림 규칙 정의
- [ ] 서비스 자동 발견 (Service Discovery)
- [ ] 통합 대시보드 구성

**예상 효과**:
- 모든 서비스의 상태를 한 곳에서 관리
- 문제 조기 대응
- 다운타임 최소화
- 운영 효율성 향상

**기술 스택**:
- Prometheus (메트릭스 수집)
- Grafana (시각화)
- Alertmanager (알림 관리)

**현재 상태**:
- LIMEN에 임시 알림 시스템 구현됨
- 향후 통합 모니터링 서비스로 마이그레이션 예정

**관련 문서**: [모니터링 전략](../04-operations/monitoring/strategy.md)
